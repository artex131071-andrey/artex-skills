# artex-skills

Репозиторий универсальных скиллов Андрея для агентных моделей и coding assistants.

Этот репозиторий является каноническим источником правды для общих скиллов, которые используются в разных проектах. Здесь хранятся сами скиллы и их краткие метаданные.

## Принципы

- **Источник истины:** В репозиторий попадают только универсальные, кросс-проектные скиллы.
- **Локализация:** Все общение, документация и комментарии к коду ведутся строго на **русском языке**.
- **Физическое копирование:** Установка выполняется путем прямого копирования файлов в глобальные директории агентов (без использования символических ссылок).
- **Чистота проектов:** Локальные дубликаты скиллов в папках проектов удаляются; используются только глобально установленные версии.

## Структура

```text
skills/
  plan-and-wait/
    SKILL.md
    meta.yaml
  ...
```

## Список скиллов

### `plan-and-wait`
Пауза перед выполнением. Описание плана и ожидание подтверждения пользователя.

### `docs-closeout`
Проверка документации (`README`, `CHANGELOG`, инструкции) при завершении задачи.

### `context7-check`
Проверка библиотек через Context7 перед установкой или обновлением.

### `review-code`
Инженерное ревью кода на баги, регрессии и соответствие стандартам.

### `readme-refresh`
Актуализация `README.md` согласно текущему состоянию проекта.

### `api-doc-sync`
Синхронизация документации API с фактическим кодом.

### `cleanup-pass`
Удаление мертвого кода, дублирования и мелких загрязнений.

### `env-drift-check`
Проверка совместимости окружений (Windows/Linux) и сценариев запуска.

### `db-change-safety`
Анализ безопасности изменений в БД и миграций.

### `gh-actions-repair`
Диагностика и починка GitHub Actions через `gh` CLI.

### `project-intake`
Первичный сбор контекста и анализ структуры проекта при входе.

## Установка (Linux)

Скиллы устанавливаются глобально для всех основных AI-агентов путем физического копирования содержимого папки `skills/` в соответствующие директории:

- **Gemini CLI**: `~/.gemini/skills/`
- **Claude Code**: `~/.claude/commands/`
- **Codex**: `~/.codex/skills/`

### Глобальные конфигурации

Помимо самих скиллов, настроены глобальные файлы инструкций, которые задают языковой стандарт и правила работы:

1. **`~/.gemini/GEMINI.md`**: содержит список всех Artex Skills и правило русского языка.
2. **`~/.claude/CLAUDE.md`**: включает три "always-on" скилла (`project-intake`, `plan-and-wait`, `docs-closeout`) и правило русского языка.
3. **`~/.codex/AGENTS.md`** и **`~/.codex/rules/global.md`**: содержат полный список скиллов и языковой стандарт.

### Использование в проектах

Для того чтобы агенты знали о наличии глобальных скиллов при работе в конкретном проекте, в корень каждого репозитория добавляются файлы:
- `GEMINI.md`
- `CLAUDE.md`
- `AGENTS.md`

Эти файлы содержат актуальный список доступных глобальных инструментов в самом верху документа.
