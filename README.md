# artex-skills

Репозиторий универсальных скиллов Андрея для агентных моделей и coding assistants.

Этот репозиторий является каноническим источником правды для общих скиллов, которые используются в разных проектах. Здесь хранятся сами скиллы и их краткие метаданные. Конкретный способ подключения в своей среде каждая модель или система определяет самостоятельно.

## Принципы

- В репозиторий попадают только универсальные, кросс-проектные скиллы.
- Проект-специфичные скиллы должны оставаться внутри соответствующих репозиториев.
- `SKILL.md` содержит поведение и правила применения скилла.
- `meta.yaml` содержит краткое описание, версию и теги.

## Структура

```text
skills/
  plan-and-wait/
    SKILL.md
    meta.yaml
  docs-closeout/
    SKILL.md
    meta.yaml
  context7-check/
    SKILL.md
    meta.yaml
```

## Список скиллов

### `plan-and-wait`

Пауза перед выполнением. Используется, когда пользователь просит сначала объяснить понимание задачи, показать план, перечислить затрагиваемые области и дождаться подтверждения.

### `docs-closeout`

Закрытие задачи через проверку документации. Используется в конце работы, чтобы проверить, достаточно ли обновлены `README`, changelog, архитектурные заметки, task docs и агентские инструкции.

### `context7-check`

Проверка зависимости до установки или обновления. Используется при выборе, сравнении или обновлении библиотек и пакетов: сначала через Context7, затем по первичным источникам.

### `review-code`

Режим инженерного ревью. Используется, когда нужно проверить изменения на баги, регрессии, архитектурные риски, пропущенные тесты и недостающие обновления документации.

### `readme-refresh`

Сверка `README` с реальным состоянием проекта. Используется после изменений в запуске, структуре, окружении, функциональности, API или сценариях разработки.

### `api-doc-sync`

Синхронизация API-документации с кодом. Используется, когда меняются endpoint-ы, контракты, схемы, payload-ы, auth-flow, интеграции или клиентские ожидания.

### `cleanup-pass`

Локальная расчистка перед закрытием задачи. Используется для удаления дублирования, мертвого кода, устаревших комментариев, naming drift и мелких структурных загрязнений без изменения поведения.

### `env-drift-check`

Проверка расхождений окружения и сценариев запуска. Используется, когда нужно убедиться, что проект одинаково предсказуемо работает на Windows и Linux, а конфиги, пути и примеры окружения не разъехались.

### `db-change-safety`

Проверка безопасности изменений в базе данных. Используется при миграциях, schema changes, SQL-правках, seed-изменениях и любых изменениях, затрагивающих данные, rollback и совместимость.

## Правило использования

Если агент или модель поддерживает локальные skills, instructions или аналогичный механизм, он должен использовать содержимое этого репозитория как источник и адаптировать подключение под свою среду самостоятельно.
