# README Refresh

Используй этот skill, когда нужно сверить `README` с реальным состоянием проекта после изменений.

## Когда применять

- менялись команды запуска, сборки, тестирования или деплоя;
- менялись переменные окружения или требования к локальной установке;
- менялась структура каталогов или точки входа;
- добавлялись, удалялись или менялись ключевые возможности проекта;
- менялся onboarding для разработчика или пользователя.

## Workflow

1. Определи, какой `README` является ближайшим и каноническим для измененной части проекта.
2. Сравни текущий код и конфигурацию с тем, что написано в `README`.
3. Проверь разделы:
   назначение, запуск, установка, environment, scripts, структура, ограничения, troubleshooting.
4. Обнови только те части, которые реально устарели.
5. Если документация должна быть разделена между несколькими `README`, укажи это явно.

## Output

Возвращай:

- какие `README` были проверены;
- какие разделы обновлены;
- что осталось потенциально устаревшим;
- если обновление не требуется, коротко объясни почему.

## Guardrails

- Не переписывай `README` целиком без необходимости.
- Не добавляй маркетинговый текст и лишнюю воду.
- Не дублируй детальную документацию в `README`, если для этого уже есть отдельные docs.
- Если реальные команды или требования не подтверждены кодом, пометь это как точку проверки.
