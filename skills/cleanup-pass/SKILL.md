# Cleanup Pass

Используй этот skill для локальной расчистки после основной реализации, когда поведение уже достигнуто и нужно убрать технический мусор вокруг измененной зоны.

## Когда применять

- задача уже решена функционально;
- рядом с измененной зоной есть дублирование, мертвый код или устаревшие комментарии;
- нужно выровнять naming, мелкие структуры и очевидные несогласованности;
- требуется небольшой cleanup без большого рефакторинга.

## Workflow

1. Ограничься локальной областью изменений и ближайшим связанным кодом.
2. Найди и убери:
   мертвый код, дубли, устаревшие комментарии, неиспользуемые импорты, явные naming drift.
3. Упростить можно только то, что не меняет контракт и ожидаемое поведение.
4. Если cleanup перерастает в рефакторинг, остановись и пометь это как отдельную задачу.

## Output

Возвращай:

- что было очищено;
- что сознательно не трогалось;
- есть ли отдельные зоны для будущего рефакторинга.

## Guardrails

- Не превращай cleanup в архитектурный рефакторинг.
- Не меняй поведение системы под видом уборки.
- Не трогай несвязанные файлы только ради “общей красоты”.
- Если нет полезного cleanup, честно скажи, что дополнительных cleanup-изменений не требуется.
